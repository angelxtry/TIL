2. Semantics

HTML은 문서를 작성하기 위한 언어다.

HTML 5.2 스펙에서 정의하는 HTML 내의 Elements, Attributes, Attribute value 들은 특정한 의미를 가지고 있으며, 이를 semantics라고 부른다. 예를 들어 ol element는 순서있는 목록을 의미하고 lang 속성은 콘텐츠의 언어를 의미한다.
이 정의는 웹브라우저, 검색 엔진과 같은 HTML 프로세서에서 허용하며, 부모 문서 및 어플리케이션에 따라 다른 컨텍스트로 제공한다.
다른 컨텍스트의 예시로 article이란 요소를 예로 들 수 있다.
이 요소는 문서 의미로 사용될 때는 주로 다른 콘텐츠와 독립적인 콘텐츠로 사용되지만 어플리케이션 의미로 사용될 때는 widget 같은 의미로 사용된다.

3. Element Definition

element는 다음의 정보를 포함한다.
  • 카테고리
    • 요소의 하위 요소 및 하위 요소로 포함되어야 하는 것의 일반적 설명
    • 카테로리를 다 외우는 것은 불가능에 가깝다.
    • HTML 유효성 검사기를 활용할 수 있다.
  • 이 요소를 사용할 수 있는 컨텍스트
    • 이 요소를 어디서 사용할 수 있는가에 대한 정보.
    • 이 정보는 이 요소를 자식으로 허용하는 요소의 콘텐츠 모델과 중복되며 편의상 제공된다.
  • 콘텐츠 모델
  • HTML에서 태그 생략: 생략하지 않는다.
  • 콘텐츠 속성

로봇 vs. 유저
HTML은 유저를 위한 것이 아니라 로봇을 위한 것
로봇: 검색엔진, 브라우저, 크롤러 같은 것들
유저: 사용자, 사이트를 방문하여 이용하는 사람
유저는 절대 유저 스스로 HTML을 해석하여 웹사이트에 뿌릴 수 없다.
로봇(브라우저)에 의해 해석된 콘텐츠를 유저가 접근하여 사용한다.
접근성: 스크린 리더와 같은 Assistant Technology(AT)가 접근 가능하게 하는 것
HTML을 잘 작성한다는 의미는 로봇이 잘 이해할 수 있게 한다는 의미다.
