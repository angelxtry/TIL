Visual formatting model

유저 에이전트가 비주얼 미디어를 위해 문서 트리를 어떻게 해석하는가? visual formatting model에서 모든 요소는 0개 혹은 그 이상의 박스를 박스 모델에 따라 생성한다. 박스의 레이아웃은 다음과 같이 구성되어 있다.

  • box dimension과 type
  • positioning scheme (normal flow, float, absolute positioning)
  • 문서 트리 내 elements 간의 관계
  • 외부 정보(viewport, 이미지의 고유한 dimension 등)

Positioning scheme

user agent에서 요소를 어떻게 배치할 지 결정하는 요인들
  • Normal flow
  • Floats
  • Absolute Positioning

Normal flow

블록 레벨 엘리면트는 블록으로 쌓이고, 인라인 레벨 엘리먼트는 인라인으로 쌓인다. CSS로 elements의 flow를 바꾸지 않은 상태

Floats

elements를 배치하는 방법 중 하나. normal flow의 첫 번째 위치로 레이아웃을 잡게 하는 것. float 속성을 넣게되는 경우 가능한 한 최대한의 left 혹은 right로 이동 float 속성은 특정한 element를 왼쪽 또는 오른쪽으로 부양시키는 역할 속성값으로 left 또는 right를 가진다.

clearfix

float element의 containing block이 제대로 높이를 가질 수 있게 하는 테크닉 .clearfix::after { content: ''; display: block; clear: both; } 형제 요소 중에 float 요소가 있다면, 그 float의 영향을 해제시켜주는 속성 clear 속성을 넣어줌으로써 float의 영향을 받지 않게 된다.

absolute positioning

position 속성을 사용하게 될 경우, containing block을 기준으로 박스를 특정 위치에 배치시키는 것이 가능 이 상태를 absolute positioning 상태라고 한다. position은 특정한 요소를 어떻게 배치할 것인가다. 지금까지 다뤘던 것은 static position이다. absolute positioning은 특정한 위치에 해당 요소를 고정시키고 싶을 때 사용한다. 해당 요소를 containing block을 기준으로 특정 위치에 고정시킬 수 있다. box offset 속성: top, right, botton, left absolute position에서 containing block은 조상 요소 중 position이 정적 position이 아닌 요소 out of flow 상태 콘텐츠 중첩 현상이 발생하기도 한다. 다른 콘텐츠의 변화에 유동적으로 대응하기 힘들다. 남발하면 안된다.
