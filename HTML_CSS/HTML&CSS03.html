HTML Tree

  • 모든 HTML은 Tree 구조
  • HTML이 내가 그릴 대상(Structure), CSS가 디테일(Graphics)
  • HTML elements는 parent 또는 child를 가질 수 있다.
  • 만약 child element를 가질 수 없다면 -> void elements(단는 태그가 없는 아이들)

문법

1. Selector

        Selector {
          Property-name:: porperty-value;
        }

Example 1: P elements의 색상을 red 

        p {
          color: red;
        }

        p {
          color: rgb(255, 0, 0);
        }

        p {
          color: #ff0000;
        }

미리 사전에 정의되어 있는 값을 키워드라고 부른다.
CSS는 미리 정의된 키워드를 이용하여 스타일을 지정할 수 있다.
CSS를 사용할 때 가장 많이 하는 행위가 키워드를 외우는 것이다.

2. CSS는 대소문자 구분을 할까?

답: Case By Case
정확한 답: 매칭 알고리즘을 사용하는 경우에는 대소문자 구분을 하고,
매칭 알고리즘을 사용하지 않는 경우(HTML 태그에 바로 접속)에는 대소문자 구분을 하지 않는다.

3. @rule

CSS에서 특수한 문법을 표현하고자 할 때 사용하는 rule

  • 웹폰트: @font-face
  • 미디어쿼리: @media
  • 애니메이션: @keyframes
  • 다른 CSS가져오기: @import(성능저하 때문에 비추)
        @media (min-width:360px) {

        }

4. BLOCKS

여는 중괄호와 닫는 중괄호 내부를 BLOCKS라고 부른다. 

        p {

        }

5. Rule Set

        /* as-is */
        h1 { color: red }
        h2 { color: red }

        /* to-be (rele set) */
        h1, h2 { color: red }

6. Declaration

        h1 { color: red }
        h1 { font-size: 16px }
        h1 { line-height: 24px }

        /* 마지막 라인에는 세미콜론이 없어도 된다. */
        h1 {
          color: red;
          font-size: 16px;
          line-height: 24px
        }
white space를 모두 제거하여도 동일하게 동작한다. 이전에는 white space를 최대한 제거하고 한 줄로 작성하는 것을 권장하기도 했지만 현재는 gulp 등의 빌드 혹은 후처리 도구를 사용하여 white space를 한번에 제거한다.
